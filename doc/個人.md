# drink-counter

> お酒の売れた数を管理・記録するためのリアルタイム・カウンターアプリ

## 📖 概要

drink-counterは、お酒の売れた数を管理・記録できるWebアプリケーションです。リアルタイム同期により、複数のデバイス間でデータが即座に反映されます。

## 🚀 セットアップ

### 1. リポジトリのクローン

```bash
git clone https://github.com/yourusername/drink-counter.git
cd drink-counter
```

### 2. 依存関係のインストール

```bash
npm install
```

### 3. Supabaseプロジェクトの作成

1. [Supabase](https://supabase.com/)でアカウントを作成
2. 新しいプロジェクトを作成
3. プロジェクトの設定から以下を取得：
   - Project URL（`https://xxxxx.supabase.co`）
   - Anon/Public Key

### 4. 環境変数の設定

`.env.example`をコピーして`.env`を作成：

```bash
cp .env.example .env
```

`.env`ファイルを編集：

```env
# Supabase設定
SUPABASE_URL=https://xxxxx.supabase.co
SUPABASE_ANON_KEY=your-anon-key-here

# Session Secret（ランダムな長い文字列を生成）
SESSION_SECRET=your-session-secret-here
```

**SESSION_SECRETの生成方法**：
```bash
# OpenSSLを使用
openssl rand -base64 32

# または Node.jsを使用
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"
```

### 5. データベースのセットアップ

Supabase CLIを使用してマイグレーションを実行：

```bash
# Supabase CLIのインストール（初回のみ）
npm install -g supabase

# プロジェクトにリンク
npx supabase link --project-ref your-project-ref

# マイグレーションの実行
npx supabase db push
```

または、Supabaseダッシュボードから手動でSQLを実行：
1. Supabaseダッシュボード → SQL Editor
2. `supabase/migrations/`内の各SQLファイルを順番に実行

### 6. 開発サーバーの起動

```bash
npm run dev
```

ブラウザで [http://localhost:5173](http://localhost:5173) を開きます。

## 🔐 認証について

このアプリケーションは**Supabase Auth**を使用しています。

### ユーザー登録

1. Supabaseダッシュボード → Authentication → Users
2. "Add user" でメールアドレスとパスワードを設定
3. または、アプリケーションで新規登録機能を実装

### 認証フロー

- ログイン時、Remixのサーバーサイドで認証を検証
- セッション情報はサーバーサイドのCookieに保存
- 認証が必要なページは`_auth`レイアウトで保護

## ⚡ リアルタイム機能

Supabaseのリアルタイム機能により、カウンターの変更が即座に他のデバイスに反映されます。

### 仕組み

1. クライアントがSupabaseのリアルタイムチャンネルに接続
2. データベースの変更をPostgreSQLのトリガーで検知
3. WebSocketを通じて全クライアントに変更を通知

### 設定

リアルタイム機能を有効化するには、Supabaseでテーブルに対してリアルタイムを有効にする必要があります：

```sql
-- supabase/migrations/enable_realtime.sql
ALTER PUBLICATION supabase_realtime ADD TABLE counters;
```

## 🛠️ トラブルシューティング

### Supabase接続エラー

**症状**: `Failed to connect to Supabase`

**解決方法**:
1. `.env`ファイルの`SUPABASE_URL`と`SUPABASE_ANON_KEY`を確認
2. Supabaseプロジェクトが起動していることを確認
3. ネットワーク接続を確認

### 認証が動作しない

**症状**: ログイン後すぐにログアウトされる

**解決方法**:
1. `SESSION_SECRET`が設定されているか確認
2. Cookieが有効になっているか確認（ブラウザの設定）
3. HTTPSを使用している場合、`secure`フラグを確認

### リアルタイムが動作しない

**症状**: カウンターの変更が他のデバイスに反映されない

**解決方法**:
1. Supabaseダッシュボード → Database → Replication
2. `counters`テーブルがリアルタイムに有効化されているか確認
3. RLSポリシーで`SELECT`権限があるか確認

### ビルドエラー

**症状**: `npm run build`が失敗する

**解決方法**:
1. `node_modules`を削除して再インストール:
   ```bash
   rm -rf node_modules package-lock.json
   npm install
   ```
2. TypeScriptエラーを確認: `npm run typecheck`
3. Node.jsのバージョンを確認（20.x以上）

## 📚 アーキテクチャ

### データフロー

```
┌─────────────┐
│   Browser   │
└──────┬──────┘
       │ 1. リクエスト
       ▼
┌─────────────────┐
│  Remix Server   │ 2. 認証チェック
│  Loader/Action  │────────────────┐
└────────┬────────┘                │
         │                         ▼
         │ 3. レスポンス    ┌─────────────┐
         │                  │  Supabase   │
         └─────────────────►│  - Postgres │
                            │  - Auth     │
         ┌──────────────────│  - Realtime │
         │ 4. WebSocket     └─────────────┘
         ▼
   ┌──────────┐
   │  Client  │
   └──────────┘
```

### セキュリティ

- **Server-side認証**: すべての認証チェックはサーバーサイドで実行
- **RLS**: SupabaseのRow Level Securityでデータアクセスを制御
- **CSRF保護**: RemixのFormコンポーネントが自動的にCSRF対策
- **環境変数**: 機密情報は環境変数で管理
